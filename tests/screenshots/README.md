# UI Screenshot Tests

This directory contains screenshots generated by `ui_rendering.rs` tests.

## Generated Screenshots

The tests automatically generate screenshots at various terminal sizes:

- `ui_80x24.txt` - Minimum size (1x1 squares)
- `ui_80x25.txt` - Slightly taller minimum
- `ui_100x30.txt` - Medium size (2x2 squares)
- `ui_120x40.txt` - Large size
- `ui_132x46.txt` - Extra large (3x3 squares)
- `ui_160x50.txt` - Very large
- `ui_200x60.txt` - Huge terminal
- `ui_narrow_80x30.txt` - Narrow layout (stacked)
- `ui_wide_150x40.txt` - Wide layout (side-by-side)
- `ui_with_moves.txt` - After making moves

## Running Tests

```bash
# Run all UI rendering tests
cargo test --test ui_rendering

# Run specific test with output
cargo test --test ui_rendering test_render_minimum_size -- --nocapture

# Generate all screenshots
cargo test --test ui_rendering test_all_standard_sizes
```

## What Tests Verify

1. **No panics** at any terminal size
2. **Board scales** appropriately (1x1, 2x2, 3x3 squares)
3. **Layout adapts** (stacked vs side-by-side)
4. **Essential elements** present (title, board, status)
5. **Checkerboard pattern** renders correctly
6. **Move history** displays after moves

## Visual Regression Testing

These screenshots can be used for visual regression testing:

1. Run tests to generate current screenshots
2. Compare with golden/reference screenshots
3. Detect unintended UI changes

## Notes

- Screenshots are plain text files
- Can be viewed in any text editor
- ANSI color codes are not captured (TestBackend limitation)
- Actual colors visible only in terminal
